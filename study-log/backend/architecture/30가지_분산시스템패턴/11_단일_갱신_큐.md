# 목적
- 여러 클라이언트/WAL가 동시에 상태를 변경/추가하려 할 때는 변경을 순차적으로(한 번에 하나씩) 처리해 일관성을 유지해야 한다
- 단일 스레드를 사용해 요청을 비동기로 처리하고 호출자 블록 없이 순서를 유지한다.

# 구현 시 주의 사항
- 공통 구조
  - 배후에 작업 큐가 존재하는 단일 스레드
  - 큐 자료구조 구현체 선택
- 채널과 경량 스레드 사용하기
  - 단일 고루틴에서 모든 요청을 처 리하고 상태를 갱신
- 배압
  - 소비자가 느리고 생산자가 빠르다면 큐가 빠르게 가득 찰 수 있다
  - 일반적으로 큐는 가득 차면 전송자를 블록해 최대 크기의 상한을 유지한다
- 여러 작업을 연쇄적으로 처리해야할 때
  - 실행의 결과를 다음 단계로 전달 필요
- 외부 서비스 호출
  - 유일한 스레드를 블록하면 안 되므로 블로킹 네트워크 호출을 하지 않아야 한다
  - 퓨처 콜백에서 단일 스레드에서 관리하는 상태값을 건들지 않아야한다.
    - 호출 결과는 다른 이벤트나 요청과 마찬가지로 작업 큐에 추가해 야 한다
