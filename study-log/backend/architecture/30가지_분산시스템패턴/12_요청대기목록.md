# 목적
다른 클러스터 노드 응답에 기반하는 기준을 만족한 후에 응답을 받아야 하는 클라이언트 요청을 추적한다.

# 문제
여러 노드로부터 응답이 동시에 들어오므로, 특정 요청이 과반수에 도달했는지 판단하기 위해 요청-응답 매핑을 관리 필요

# 해결
- 클러스터 노드는 키와 콜백을 매핑한 대기 목록(waiting list)을 유지하며, 비동기 응답을 관리한다.
- 각 요청에는 상관 ID(correlation ID) 같은 키가 붙고, 이를 통해 응답을 해당 요청과 연결한다.
-  노드는 다른 클러스터 노드에 요청을 보낼 때마다 콜백을 등록하고, 응답이 도착하면 대기 목록을 통해 해당 콜백을 호출한다.
- 콜백은 응답을 처리하고, 과반수 정족수(Quorum)에 도달했는지 판단하여 클라이언트에게 응답할지 결정한다.

# 추가 고민 사항
- 때로는 다른 클러스터 노드로부터 응답이 지연된다.
- 이 경우를 대비해 대기 목록은 타임아웃이 지난 후 요청을 만료하는 메커니즘이 필요하다.
