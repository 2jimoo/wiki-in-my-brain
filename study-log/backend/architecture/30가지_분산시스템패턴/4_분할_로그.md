# 문제
- 단일 로그 파일은 점점 커지다 보면 기동할 때 이를 읽는 과정에서 성능 병목이 생길 수 있다
- 오래된 로그는 주기적으로 지우지만，거대한 단일 파일을 지우는 작업은 구현하기가 쉽지 않다
- 하나의 큰 파일 대신 작은 크기의 여러 파일로 나누어 작업을 쉽게 한다

# 롤링(롤오버)
- 논리 로그 오프셋(또는 로그 시권스 번호)을 로그 분할 파일에 매핑 연산
  - 각 로그 분할의 이름을 정해 진 방식의 접두사(prefix)와 기본 오프셋(시퀀스 번호)으로 생성
  - 각각의 로그 시퀀스 번호를 파일 이름과 트랜잭션 오프셋 두 부분으로 나눈다
- 읽기 연산
  - 주어진 오프셋(또는 트랜잭션 ID)에서 로그 분할을 식별하고 후속 로그 분할에서 로그 레코드를 모두 읽는다
