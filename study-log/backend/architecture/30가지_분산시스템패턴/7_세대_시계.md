# 목적
- 서버 세대를 나타내는 단조 증가하는 숫자. 텀(Term)，에폭(Epoch), 세대(Generation) 모두 같은 말
- 기존 리더가 잠깐 응답 안 한 사이 새 리더 선출되었을 때

# 램포트 시계
- 스템 시계에 의존하지 않고 프로세스 집합 전반에 걸친 이벤트 순서를 결정하는 기법
 - 각각의 프로세스는 작업을 수행할 때마다 정수 카운터를 증가시킨다 
 - 프로세스 사이에서 메시지를 교환할 때 이 정숫값을 다른 프로세스에 보낸다
 - 메시지를 받은 프로세스는 자신의 카운터 값과 메시지의 정숫값 중 큰 값으로 자신의 정수 카운터를 설정한다
 - 연관된 정숫값을 비교해 어떤 작업이 다른 작업보다 먼저 발생했는지 알수 있다
 - 이렇게 비교할 수 있는 작업들은 인과 관계(causallyrelated)가 있다고 말한다
- 세대 시계는 램포트 시계의 일종

# 어떻게 팔로워로 퇴출?
- 리더가 바뀔 때마다 세대를 올리고, 그 세대 번호를 메시지·로그에 계속 실어서 클러스터 전체가 “지금은 몇 번째 리더 세대인지”를 공유
- 요청에 들어 있는 세대값이 자기 세대보다 작으면 “너는 예전 세대의 리더/요청이다”라고 판단하고 실패 응답을 보낸다
- 응답 세대가 자기보다 크면 자기 세대를 갱신하고 리더 자리에서 물러나 팔로워로 전환한다.
