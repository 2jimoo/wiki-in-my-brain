# 목적
- 일부 노드들 간에 연결이 끊어졌을 때도 전체 클러스터가 안전하게 합의에 도달
- 단일 값 특화

# 개요
- 준비 단계
  - 제안자 노드가 모든 수락자 노드에게 자신의 (최신 세대 시계, 제안 값)을 받아줄 건지 물어봄
  - 단일 실패 지점을 예방하기 위해 각ㄱ각의 노드는 세대번호+노드ID로 자체 세대 시계 유지
- 수락 단계
  - 세대 시계 갱신해서 수락 요청 반환
  - 허락한 세대번호 이전 요청들은 거절/무시
  - 다른 제안자가 정족수를 달성하려면 이전 과반수 정족수 멤버 둥 최소 하나의 멤버 이탈 필요
    - 두 개 이상의 제안자가 순환에 빠지는 라이브락 발생 가능(지연 등으로 완화)
- 커밋 단계
  - 과반수가 수락하면 합의 완료 
  - 제안자가 이 선택된 값을 클러스터의 모든 노드에게 커밋
